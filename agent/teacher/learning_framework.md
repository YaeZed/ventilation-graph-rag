# 📚 矿井通风智能辨识系统：3天源码学习框架

本指南基于 `data_pipeline`（数据管道）和 `rag_system`（检索增强生成系统）两大核心模块，为你梳理了从零开始掌握该系统的 3 天硬核学习大纲。

建议按照以下顺序阅读源码，结合本地运行和日志输出，可以达到最佳学习效果。

---

## 🗓️ Day 1：数据管道与知识图谱构建（Data Pipeline）

**学习目标**：理解系统如何把非结构化的《煤矿安全规程》Word 文档，变成 Neo4j 里面的结构化节点和关系（Article, Requirement, Parameter, Facility, Location 等）。

**📍 核心代码目录**：`agent/data_pipeline/`

| 学习顺序          | 核心文件                                                     | 学习重点与思考题                                                                                                                      |
| :---------------- | :----------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------ |
| **1. 整体入口**   | `run_ventilation_agent.py`                                   | 代码的执行起点。重点看它如何协调整个大模型的抽取流程，了解它是如何按批次（Batch）处理文档条目的。                                     |
| **2. 知识建模**   | `ventilation_kg_builder.py`<br>`ventilation_safety_agent.py` | 思考：Agent 是如何让大模型从文本中抽取出**实体**的？重点看 Prompt 设计，以及它是怎么定义知识图谱架构（Schema）的。                    |
| **3. 数据清洗**   | `amount_normalizer.py`<br>`safety_logic_normalizer.py`       | 看看大模型抽取出的脏数据（比如单位不统一、逻辑有歧义的上下限）是如何通过正则或逻辑规则被清洗标准化的。                                |
| **4. 批次管理**   | `batch_manager.py`                                           | 因为文本太长，代码如何进行断点续传和分批（Batch）调用的？                                                                             |
| **5. 入库 Neo4j** | `import_to_neo4j.py`                                         | **重点**：学习 `cypher` 语句。看代码是如何将 CSV 或内存中的抽取结果，通过 `MERGE`、`MATCH` 等指令真正写入 Neo4j，并生成边（关系）的。 |

**💡 实践建议**：在看 `import_to_neo4j.py` 时，打开 Neo4j Browser (`localhost:7474`)，查一条数据（例如 `MATCH (n) RETURN n LIMIT 25`）看看真实图谱长什么样，这比干看代码有用的多。

## 🗓️ Day 2：混合检索与图索引底层机制（Retrieval Foundation）

**学习目标**：搞懂当用户问一个问题时，系统是如何在茫茫数据中找到最相关的规程条款的。这是 GraphRAG 最核心的难点。

**📍 核心代码目录**：`agent/rag_system/`

| 学习顺序            | 核心文件                             | 学习重点与思考题                                                                                                                                                                                |
| :------------------ | :----------------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **1. 数据准备**     | `ventilation_data_preparation.py`    | 这是衔接 Day1 的桥梁。关注 `build_article_documents()` 方法，看它是如何把 Neo4j 里的零散节点（设施、要求、指标）拼装成一个**富文本文档**，用来喂给 Milvus 等向量数据库的。                      |
| **2. 图索引机制**   | `ventilation_graph_indexing.py`      | GraphRAG 的精髓。学习它如何将图谱结构转换为键值对索引（Key-Value Index），以便后续做 Entity-level（实体级）和 Topic-level（主题级）的图谱专属检索。                                             |
| **3. 传统混合检索** | `ventilation_hybrid_retrieval.py`    | 重点理解**传统的双路检索范式**：代码是如何把 向量检索（Milvus语义搜索）与 关键词检索（BM25字典匹配或图节点查询）结合起来进行召回合并的？                                                        |
| **4. 高阶图检索**   | `ventilation_graph_rag_retrieval.py` | **核心重头戏**：这部分做了**多跳推理遍历**（Multi-hop Reasoning）和**知识子图提取**。重点看代码是如何根据一个初始节点，向外扩展 1～2 跳（Neighbor Expansion），从而找到隐藏在深处的关联条款的。 |

## 🗓️ Day 3：智能路由、流水线与答案生成（Pipeline & Generation）

**学习目标**：掌握流水线的整体组装。观察系统如何判断问题难度，动态选择最佳检索策略，并最终生成具有专业逻辑的解答。

**📍 核心代码目录**：`agent/rag_system/`

| 学习顺序          | 核心文件                                                     | 学习重点与思考题                                                                                                                                              |
| :---------------- | :----------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **1. 智能路由**   | `智能查询路由器`<br>_(参考原版 intelligent_query_router.py)_ | 思考：系统怎么知道一个问题该用简单混合检索还是高级图 RAG 检索？如何评估查询的"复杂度"（比如包含几个实体、是否需要关系推理）？这对降低大模型调用成本至关重要。 |
| **2. 答案生成**   | `ventilation_generation.py`                                  | 看看它是怎么构造最终丢给大模型（如 Qwen）的 Prompt 的。关注其规定的输出格式：结论、依据条款、分析、整改建议。这是"智能辨识专家"大脑的出口。                   |
| **3. 主流水线**   | `ventilation_rag_pipeline.py`                                | **大满贯**：看它是如何把 Day2 中零散的检索器、生成器实例化，并串联在一起的。这是整个系统对外提供服务的 API/核心类。                                           |
| **4. 测试与验证** | `ventilation_test_queries.py`<br>`test_results.md`           | 用测试脚本跑几个实际的 Query。看看日志输出中检索到了哪些片段，生成了怎样的回答，对照 `test_results.md` 中的效果去理解代码为什么要这么写。                     |

## 🎓 学习心得与进阶提示

1. **抓大放小**：第一遍不要死扣每一行具体的逻辑（比如正则的写法、复杂的 API 参数）。重点看**类名、函数名、输入和输出**。掌握数据流向（如：`输入 String -> 转换为 List[Document] -> Embedding 转换为 Vector`）。
2. **结合日志运行（Log Driven Learning）**：阅读完源码后，一定要打开日志（DEBUG 级别）运行 `ventilation_test_queries.py`。一边看着终端打印出来的详细 LOG，一边对着代码看它走到了哪个函数的哪个分支。
3. **带着验证的视角**：思考如果你需要在系统中新增一类实体（比如："违规处罚"），在这 3 天的流程中，你需要修改哪些代码文件？
